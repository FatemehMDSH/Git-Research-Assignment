دستور git pull برای دریافت و دانلود محتوا از یک مخزن راه دور (Remote) و به‌روزرسانی بلافاصله مخزن محلی (Local) استفاده می‌شود. این دستور یکی از پرکاربردترین دستورات برای هماهنگ نگه داشتن پروژه شخصی با تغییراتی است که دیگران روی مخزن اصلی انجام داده‌اند.

این دستور در پشت صحنه (Under the hood) چگونه کار می‌کند؟
از نظر فنی، دستور git pull ترکیبی از دو دستور مجزا است:

۱. git fetch: این دستور محتوا را از مخزن ریموت می‌خواند و در مخزن محلی شما (جایی که HEAD به آن اشاره می‌کند) دانلود می‌کند، اما تغییری در کدهای فعلی شما ایجاد نمی‌کند.

۲. git merge: این دستور تغییرات دانلود شده را با شاخه فعلی شما ترکیب می‌کند. در این مرحله یک کامیت جدید ایجاد شده و HEAD برای اشاره به این کامیت جدید به‌روزرسانی می‌شود.

خلاصه فرآیند:
Fetch: داده‌های جدید را دانلود می‌کند اما آن‌ها را با فایل‌های در حال کار شما ترکیب نمی‌کند.

Merge: تغییرات واکشی شده را در شاخه فعلی شما ادغام می‌کند.

نتیجه نهایی: دایرکتوری کاری شما کاملاً با نسخه موجود در سرور (Remote) همگام و به‌روز می‌شود.
