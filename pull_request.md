(Pull Request) یکی از مهم‌ترین قابلیت‌های گیت‌هاب برای همکاری‌های تیمی است. زمانی که شما تغییری را در یک مخزن (مثلاً در نسخه Fork شده خودتان) ایجاد می‌کنید، با ارسال یک Pull Request به صاحب پروژه اصلی اطلاع می‌دهید که کدهای جدیدی آماده کرده‌اید و از او می‌خواهید که این تغییرات را بررسی و در صورت تایید، با پروژه اصلی ادغام (Merge) کند.

در پشت صحنه (Under the hood) چه اتفاقی می‌افتد؟
وقتی شما یک PR ایجاد می‌کنید، برخلاف دستوراتی مثل push یا pull، اتفاق فنی فوری در فایل‌های مخزن اصلی نمی‌افتد، بلکه فرآیندهای زیر در دیتابیس گیت‌هاب طی می‌شود:

۱. ایجاد یک تالار گفتگو: گیت‌هاب یک صفحه اختصاصی برای درخواست شما باز می‌کند که در آن تمام تغییرات (Diffs) بین کدهای شما و پروژه اصلی نمایش داده می‌شود.

۲. بررسی تداخل‌ها (Conflict Check): گیت‌هاب به صورت خودکار چک می‌کند که آیا کدهای شما با آخرین نسخه پروژه اصلی تداخل دارد یا خیر.

۳. کد ریویو (Code Review): سایر برنامه‌نویسان می‌توانند روی خط به خط کدهای شما نظر بدهند یا از شما بخواهند اصلاحاتی انجام دهید.

۴. ادغام نهایی (Merge): پس از تایید، صاحب پروژه دکمه Merge را می‌زند. در این لحظه گیت‌هاب دستور git merge را بین مخزن شما و مخزن اصلی اجرا می‌کند و یک کامیت جدید در پروژه اصلی ثبت می‌شود.

چرا Pull Request مهم است؟
کنترل کیفیت: اجازه می‌دهد قبل از ورود هر کدی به پروژه اصلی، توسط دیگران بازبینی شود.

همکاری بدون دسترسی مستقیم: شما می‌توانید برای پروژه‌های بزرگ دنیا (مثل لینوکس یا ری‌اکت) کد بنویسید، بدون اینکه از قبل اجازه دسترسی مستقیم به مخزن آن‌ها را داشته باشید.

مستندسازی: تاریخچه تمام بحث‌ها و دلیل تغییرات برای همیشه در گیت‌هاب باقی می‌ماند.
